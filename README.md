# DIY谐波赤道仪（已通电验证，待实拍测试）

由于我厌倦了EQ3D赤道仪的沉重，我计划用更轻便的谐波赤道仪来替换它，然而市面上的谐波赤道仪价格太贵（与DIY的规格相比），所以这个项目就此诞生，谐波赤道仪能够在不牺牲性能的前提下，提供高精度、便携性和经济性

# 项目描述

自制谐波赤道仪的核心谐波减速器，是基于Leaderdriver拆机二手的工业谐波减速器(也适用于淘宝购买的同规格谐波减速器如“25-100”)

该项目旨在解决业余天文设备中的关键问题：
- 成品支架价格高昂
- 缺乏开放性和灵活性
- 复杂性或与个人设置的不兼容

此款DIY谐波赤道仪优势是：
- 🧰**经济实惠**——总造价低于3000元，加工零件价格低于1500元
- 🪶**相对便携**——不含底座重量7.5公斤
- 🏗️**易于搭建**——专为拥有基本工具的创客设计
- 💪**高性能**——精确跟踪（亚角秒级 RMS），无需配重
- ⚠️**关于当前项目状态的说明**⚠️
该项目只完成了通电和简单的goto测试未进行实拍测试

## 🔌 电子部分概览

控制系统完全采用定制设计，并围绕紧凑结构构建，它包括：

* **基于 ESP32-WROOM-32E 的主控核心**，运行强大的 [OnStep](https://onstep.groups.io/g/main/wiki) 系统，提供高效的无线/有线通讯
* **TMC系列步进电机驱动器**，实现赤经与赤纬轴精确、静音的微步进控制与指向
* **高效电源管理方案**，板载 TPS5430 降压模块和 AMS1117 LDO 稳压芯片，确保高负载下系统电压的纯净与稳定
* 集成 **零位传感器和限位开关** 接口，在运行中保障设备的安全
* 支持通过 Wi-Fi、蓝牙或 USB 路由进行远程控制，无缝对接主流天文摄影软件生态 (N.I.N.A., INDI, ASCOM 等)

所有电路板原理图及 PCB Layout（严格把控了安全间距、过孔尺寸和大电流线宽）均使用开源工具设计，您可以在 [https://member.jlc.com/?spm=PCB.Homepage] 订购 PCB或对其进行重新组合以满足您的需求

## 💻 固件配置与刷写

请参阅 [Firmware_Setup.md](Firmware_Setup.md) 获取完整的 `Config.h` 参数表。

Dual 25 谐波赤道仪的主控大脑基于 ESP32-WROOM-32E 模块，并运行高度可定制的开源 [OnStep](https://onstep.groups.io/g/main/wiki) 固件。由于我们采用了定制的 PCB 布局和 25-100 谐波减速器，在刷写前需要对固件参数进行精确适配。

### ⚙️ 准备工作与参数配置

1. **获取源码与环境**：从官方渠道下载最新稳定版的 OnStep 源码。推荐使用 Arduino IDE 或 PlatformIO 进行编译，并确保已安装最新的 ESP32 开发板支持包。
2. **核心参数调整 (`Config.h`)**：
   * **引脚映射 (Pinmap)**：选择或自定义与本 PCB 匹配的 ESP32 引脚定义文件，确保电机控制、电源管理及各传感器接口对应正确。
   * **传动比计算**：这是最关键的一步。请根据 25-100 谐波减速比、步进电机步距角及 TMC 驱动器的细分设置，精确计算并填入赤经 (RA) 和赤纬 (Dec) 的每度步数 (`AXIS_STEPS_PER_DEGREE`)。
   * **驱动器配置**：启用 TMC 驱动器支持，调整电流参数，并根据观测需求配置静音步进模式以降低运行噪音。
   * **安全与寻星**：由于谐波减速器具有极高的扭矩，**务必启用并准确配置 Homing（寻星）和限位开关 (Limit Switches)**，以保护机械结构免受意外干涉。

### 🚀 刷写步骤

1. **连接主控板**：使用优质的 USB 数据线将定制控制板连接到电脑。
   * *注：系统通讯依赖板载的 CP2102N 串口芯片，请确保您的电脑已安装最新的 Silicon Labs CP210X 驱动程序。*
2. **编译与上传**：在开发环境中选择对应的 COM 端口和 `ESP32 Dev Module`，点击编译并上传。
   * *故障排除：如果在上传时遇到 `Connecting...` 超时错误，请在终端出现该提示时，长按 PCB 上的 `BOOT` 按钮，直到进度条开始滚动。*

> ⚠️ **安全警告：测试与校验**
> 在首次通电且挂载昂贵的光学设备之前，请**务必进行空载测试**。通过 OnStep Web 界面或手机 App 移动各轴，仔细验证电机的运转方向、限位开关的触发状态以及基础的指向模型 (Pointing Model) 是否正常工作。强烈建议在确认一切无误后再进行实战观测！
